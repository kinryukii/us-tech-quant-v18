V18.45B OPERATOR REFRESH COMMAND SUMMARY
========================================

目的
----
本文档用于记录 V18.45B 之后的日常操作指令。

当前系统支持两种刷新模式：

1. Rolling Refresh
   - 日常省时间刷新
   - 适合每天观察候选
   - 速度更快
   - 可能保留部分 stale price rows

2. Full Refresh
   - 严谨全量刷新
   - 适合买入前、大行情后、重大事件后
   - 会执行 full universe recompute + freshness audit
   - 如果价格没有全部刷新到最新，会明确标出 stale tickers

安全状态
--------
以下状态必须保持不变：

AUTO_TRADE: DISABLED
AUTO_SELL: DISABLED
TRADING_EXECUTION_ALLOWED: FALSE
BROKER_API_USED: FALSE
ORDER_EXECUTION_USED: FALSE

V18.45B 当前确认：

RANKING_LOGIC_CHANGED: FALSE
FACTOR_WEIGHTS_CHANGED: FALSE

1. 日常省时间刷新：Rolling
-------------------------

适合：
- 每天看盘
- 只是观察候选
- 不准备马上买入
- 想节省时间

推荐命令：

等价命令：

2. 严谨全量刷新：Full
--------------------

适合：
- 准备买入前
- 大跌大涨后
- 财报、FOMC、CPI、NVDA 等重大事件后
- TopN 里 stale 太多时
- 想重新全量重算候选池时

推荐命令：

powershell -NoProfile -ExecutionPolicy Bypass -File "D:\us-tech-quant\scripts\v18\run_v18_daily_full_refresh.ps1"

等价命令：

powershell -NoProfile -ExecutionPolicy Bypass -File "D:\us-tech-quant\scripts\v18\run_v18_current_daily_command_center.ps1" -RefreshMode Full

9. V18.45B 后的系统理解
-----------------------

Rolling:
- 日常快速观察
- 省时间
- 可接受部分数据不是最新
- 系统会用 freshness gate 标出 stale rows

Full:
- 买入前或重大行情后使用
- 执行 full universe recompute
- 执行 freshness audit
- 如果仍存在 stale rows，会明确报告

当前规则：
- stale TopN 不会被当作 actionable
- Homepage 会显示 refresh mode、stale TopN、trust level
- 交易仍然完全关闭
